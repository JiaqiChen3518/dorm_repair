这是一份周日匆匆忙忙赶出来的周记哈哈，平时的一些想法和感悟我其实都写在 README 里了，后续我会认真写周记的嘻嘻。

周一早上知道这周的任务后，我当天晚上就开动了，说实话还挺兴奋的——之前刷的那套带框架的课程看了 100 多集，终于能派上用场了！接下来几天我恶补 Spring Boot、MyBatis、会话跟踪 JWT、文件上传、RESTful 这些知识，重新巩固一下，想着总算能把它们真正用起来了。学着学着，发现和之前学的 Tomcat、Servlet 也能串到一起，多亏寒假做的银行管理系统打底，我很快就过渡到了 Spring Boot，真心被框架的便利和灵活惊艳到了。

差不多到周四，我把最基本的 API 接口、业务逻辑、Mapper 层的 SQL 语句，还有前端的按钮和输入框都搭了出来。周五周六嘛，基本就是和 AI 疯狂修补的两天 😂😂。

不得不说，AI 在改代码这件事上真的太细致了。有些功能我自己还摸不着头脑，AI 就直接帮我搞定了。这周一个比较大的改动，是用前端的 localStorage 配合后端的 JWT 令牌校验，替换掉原来的 session 机制。一开始登录完再退出，重新进页面就卡住了，查了半天才发现是前端 localStorage 里的用户信息没清掉，后端服务器重启后又不保留之前的用户信息，前后端就这么断开了。后来 AI 帮我加了一个响应拦截器（才知道前端也有拦截器），当访问未登出的页面时，后端返回 401 状态码，前端拦截器一收到就自动把 localStorage 里的用户信息清掉，直接退回登录页。这种细节处理真的是我自己写的时候完全没想到的。

还有常量命名规范也让我印象深刻。我一开始就是直接在类里声明常量，后面发现其他类要用的时候到处都在找，零零散散堆在各个地方。AI 帮我把相关常量统一抽到一个类里，做成静态常量，直接用类名就能调，一下子清爽了好多。

说实话，我本来觉得自己的 MySQL 水平还行，基本的增删改查、条件查询、模糊查询之前在银行管理系统里也练得挺熟了，但 AI 写的 SQL 是真的规范，挑不出毛病。业务逻辑层也是，我自认为对数据的处理和判断已经考虑得挺全了，AI 介入之后又把各种边边角角的错误全给补上了。

这周除了自己搭基本功能，其他时间几乎都在和 AI 边改边学。新收获也挺多的：JWT 令牌校验、Spring Boot、MyBatis、阿里云文件上传、拦截器（一开始写的过滤器，后来换成拦截器了）等等，之前学的 JavaWeb 知识也顺带巩固了一遍，还跟着学了不少代码规范和命名规范。后续我打算再添点功能，继续拿新学的东西把这个项目打磨得更好。

# Web 后端实战 - 员工管理笔记

## 一、员工列表查询（实战）

核心需求：实现员工信息的**分页查询**、**条件分页查询**，关联部门表查询部门名称，使用三层架构 + MyBatis 实现。

### 1. 准备工作

1. 准备数据库表：`emp`（员工表）、`dept`（部门表）；
2. 编写实体类：`Emp`（含部门名称`deptName`字段）、`PageResult`（分页结果封装类）；
3. 搭建三层架构：`EmpController`、`EmpService/EmpServiceImpl`、`EmpMapper`；
4. 开启 MyBatis**驼峰命名自动映射**（解决数据库`xxx_yyy`与实体类`xxxYyy`字段映射）。

### 2. 分页结果封装类`PageResult`

后端需返回**总记录数**和**当前页数据列表**，统一封装为对象返回前端：

```java
@Data
public class PageResult<T> {
    private Long total; // 总记录数
    private List<T> rows; // 当前页数据列表
}
```

### 3. 基础分页查询

#### （1）核心参数

- 前端传递：`page`（当前页码，默认 1）、`pageSize`（每页记录数，默认 10）；
- 后端处理：分页起始索引`start = (page - 1) * pageSize`（MySQL`limit`的第一个参数）。

#### （2）MySQL 分页语法

```java
-- 查询当前页数据：limit 起始索引, 每页记录数
select * from emp e left join dept d on e.dept_id = d.id limit 0,10;
-- 查询总记录数
select count(*) from emp e left join dept d on e.dept_id = d.id;
```

#### （3）原始方式实现（三层架构）

##### Controller 层：接收参数，调用 Service

```java
@RestController
@RequestMapping("/emps")
public class EmpController {
    @Autowired
    private EmpService empService;

    @GetMapping
    public Result page(
        @RequestParam(defaultValue = "1") Integer page,
        @RequestParam(defaultValue = "10") Integer pageSize
    ) {
        PageResult<Emp> pageResult = empService.page(page, pageSize);
        return Result.success(pageResult);
    }
}
```

**重点**：`@RequestParam(defaultValue = "1")`为参数设置**默认值**，避免前端不传参报错。

##### Service 层：调用 Mapper，封装分页结果

```java
@Service
public class EmpServiceImpl implements EmpService {
    @Autowired
    private EmpMapper empMapper;

    @Override
    public PageResult<Emp> page(Integer page, Integer pageSize) {
        // 1. 查询总记录数
        Long total = empMapper.count();
        // 2. 计算起始索引，查询当前页数据
        Integer start = (page - 1) * pageSize;
        List<Emp> empList = empMapper.list(start, pageSize);
        // 3. 封装并返回
        return new PageResult<>(total, empList);
    }
}
```

##### Mapper 层：编写 SQL，实现数据访问

```java
@Mapper
public interface EmpMapper {
    // 查询总记录数
    @Select("select count(*) from emp e left join dept d on e.dept_id = d.id")
    Long count();

    // 查询当前页数据列表
    @Select("select e.*, d.name deptName from emp e left join dept d on e.dept_id = d.id limit #{start},#{pageSize}")
    List<Emp> list(Integer start, Integer pageSize);
}
```

### 4. PageHelper 分页插件（**推荐使用**）

#### （1）插件介绍

MyBatis 的第三方分页插件，**简化分页操作**，无需手动计算起始索引和查询总记录数，底层自动生成`count(*)`和`limit`语句。

#### （2）使用步骤

##### 1. 引入依赖（SpringBoot Starter）

```xml
<dependency>
    <groupId>com.github.pagehelper</groupId>
    <artifactId>pagehelper-spring-boot-starter</artifactId>
    <version>1.4.7</version>
</dependency>
```

##### 2. Mapper 层：编写**普通查询 SQL**，无需考虑分页

```java
@Mapper
public interface EmpMapper {
    // 仅查询所有数据，PageHelper自动分页
    @Select("select e.*, d.name deptName from emp e left join dept d on e.dept_id = d.id order by e.update_time desc")
    List<Emp> list();
}
```

##### 3. Service 层：设置分页参数，解析分页结果

```java
@Override
public PageResult<Emp> page(Integer page, Integer pageSize) {
    // 1. 设置分页参数（紧跟的第一条SQL会被分页）
    PageHelper.startPage(page, pageSize);
    // 2. 执行查询，返回结果为Page对象（继承ArrayList）
    List<Emp> empList = empMapper.list();
    // 3. 强转为Page对象，获取总记录数和当前页数据
    Page<Emp> pageInfo = (Page<Emp>) empList;
    // 4. 封装返回
    return new PageResult<>(pageInfo.getTotal(), pageInfo.getResult());
}
```

#### （3）核心原理 & 注意事项

1. 原理：PageHelper 自动拦截 MyBatis 执行，为紧跟的第一条 SQL 拼接`count(*)`（查总记录数）和`limit`（查分页数据）；
2. 注意：
   - SQL 语句结尾**不要加分号 (;)**，否则分页失效；
   - `PageHelper.startPage()`**仅对紧跟的第一条 SQL**生效；
   - 返回的`List`实际为`Page`对象，可强转获取分页信息。

### 5. 条件分页查询

#### （1）查询条件

前端支持按**员工姓名（模糊）**、**性别**、**入职时间范围**查询，结合分页参数一起传递。

#### （2）核心技术点

1. 日期参数接收：`@DateTimeFormat(pattern = "yyyy-MM-dd")`指定前端日期格式；
2. 多参数封装：参数较多时封装为**查询参数对象**，简化 Controller 方法签名；
3. 动态 SQL：使用 MyBatis 的`<if>`、`<where>`标签实现**动态拼接查询条件**。

#### （3）实现步骤

##### 1. 封装查询参数对象`EmpQueryParam`

```java
@Data
public class EmpQueryParam {
    private Integer page = 1; // 默认页码1
    private Integer pageSize = 10; // 默认每页10条
    private String name; // 员工姓名
    private Integer gender; // 性别
    @DateTimeFormat(pattern = "yyyy-MM-dd")
    private LocalDate begin; // 入职开始时间
    @DateTimeFormat(pattern = "yyyy-MM-dd")
    private LocalDate end; // 入职结束时间
}
```

##### 2. Controller 层：接收参数对象

```java
@GetMapping
public Result page(EmpQueryParam param) {
    PageResult<Emp> pageResult = empService.page(param);
    return Result.success(pageResult);
}
```

##### 3. Mapper 层：使用**动态 SQL**编写条件查询（XML 方式）

创建`EmpMapper.xml`，使用`<if>`判断条件，`<where>`自动处理多余`and/or`：

```java
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper
        PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="com.itheima.mapper.EmpMapper">
    <select id="list" parameterType="com.itheima.pojo.EmpQueryParam" resultType="com.itheima.pojo.Emp">
        select e.*, d.name as deptName
        from emp e
        left join dept d on e.dept_id = d.id
        <where>
            <!-- 姓名模糊查询：判断姓名非空且非空字符串 -->
            <if test="name != null and name != ''">
                e.name like concat('%',#{name},'%')
            </if>
            <!-- 性别精确查询：判断性别非空 -->
            <if test="gender != null">
                and e.gender = #{gender}
            </if>
            <!-- 入职时间范围：判断开始和结束时间都非空 -->
            <if test="begin != null and end != null">
                and e.entry_date between #{begin} and #{end}
            </if>
        </where>
        order by e.update_time desc
    </select>
</mapper>
```

##### 4. Service 层：结合 PageHelper 实现条件分页

```java
@Override
public PageResult<Emp> page(EmpQueryParam param) {
    // 设置分页参数
    PageHelper.startPage(param.getPage(), param.getPageSize());
    // 执行条件查询
    List<Emp> empList = empMapper.list(param);
    // 解析分页结果
    Page<Emp> pageInfo = (Page<Emp>) empList;
    return new PageResult<>(pageInfo.getTotal(), pageInfo.getResult());
}
```

### 6. MyBatis 动态 SQL（`<if>`、`<where>`）

#### （1）使用场景

SQL 语句随用户输入 / 外部条件**动态变化**时使用（如条件分页查询、多条件筛选）。

#### （2）核心标签

|   标签    |                             作用                             |
| :-------: | :----------------------------------------------------------: |
|  `<if>`   | 条件判断，`test`属性写判断表达式，条件为`true`时拼接标签内的 SQL 片段 |
| `<where>` | 自动生成`where`关键字，**自动去除条件前多余的`and`/`or`**，无条件时不生成`where` |

#### （3）关键注意点

- `<if>`的`test`属性中，判断字符串非空需同时判断`!= null`和`!= ''`；
- 条件片段前的`and`/`or`可大胆写，`<where>`会自动处理多余的连接符；
- 模糊查询使用`concat('%',#{参数},'%')`，避免 SQL 注入。

## 二、核心知识点总结

### 1. 分页查询

- 原始方式：手动计算`start = (page-1)*pageSize`，分别查总记录数和分页数据；
- PageHelper：简化分页，三步操作「引依赖→写普通 SQL→设置分页参数」；
- 分页结果：必须返回`total`（总记录数）和`rows`（当前页数据）。

### 2. 条件查询核心

- 参数接收：日期用`@DateTimeFormat`指定格式，多参数封装为**查询对象**；
- 动态 SQL：`<if>`判断条件，`<where>`处理连接符，实现动态条件拼接；
- 模糊查询：使用`concat('%',#{param},'%')`，杜绝直接拼接字符串引发 SQL 注入。

### 3. 注解使用

|         注解          |                             作用                             |
| :-------------------: | :----------------------------------------------------------: |
|    `@RequestParam`    | 接收请求参数，设置默认值（`defaultValue`），解决参数名不一致问题 |
|   `@DateTimeFormat`   |  接收日期类型参数，指定前端传递的日期格式（如`yyyy-MM-dd`）  |
|       `@Mapper`       |    标识 MyBatis 的 Mapper 接口，由 Spring 自动创建实现类     |
| `@Select`/`@Delete`等 |  MyBatis 注解式 SQL，简单 SQL 使用，复杂条件用 XML 动态 SQL  |

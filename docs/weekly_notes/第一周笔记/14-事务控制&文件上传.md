# Web 后端实战 - 员工管理（进阶）笔记

## 一、新增员工功能

### 1. 需求分析

需同时保存两类数据，保证数据一致性：

- 员工基本信息（存储到`emp`表）
- 员工工作经历（批量存储到`emp_expr`表）
- 核心关联：工作经历表通过`emp_id`关联员工表主键`id`

### 2. 核心技术点

- 主键回显：插入员工后获取自增主键，用于关联工作经历
- 批量插入：通过 MyBatis 动态 SQL`<foreach>`实现工作经历批量保存
- 事务控制：保证员工信息和工作经历要么同时成功，要么同时失败

### 3. 实现步骤

#### （1）实体类设计

`Emp`类包含工作经历列表（`exprList`），建立一对多关联：

```java
@Data
public class Emp {
    private Integer id;
    private String username;
    private String name;
    private Integer gender;
    // 其他基本属性（phone、job、salary等）
    private LocalDateTime createTime;
    private LocalDateTime updateTime;
    private List<EmpExpr> exprList; // 关联工作经历列表
}

@Data
public class EmpExpr {
    private Integer id;
    private Integer empId; // 关联员工ID
    private String company;
    private String job;
    private LocalDate begin;
    private LocalDate end;
}
```

#### （2）Mapper 层实现

##### 1. 员工基本信息插入（主键回显）

通过`@Options`注解获取自增主键，赋值给`Emp`对象的`id`属性：

```java
@Mapper
public interface EmpMapper {
    @Insert("insert into emp(username, name, gender, phone, job, salary, image, entry_date, dept_id, create_time, update_time) " +
            "values (#{username},#{name},#{gender},#{phone},#{job},#{salary},#{image},#{entryDate},#{deptId},#{createTime},#{updateTime})")
    // useGeneratedKeys=true：启用主键回显；keyProperty="id"：回显主键赋值给Emp的id属性
    @Options(useGeneratedKeys = true, keyProperty = "id")
    void insert(Emp emp);
}
```

##### 2. 工作经历批量插入（动态 SQL`<foreach>`）

通过`<foreach>`遍历`exprList`集合，拼接批量插入 SQL：

```java
<!-- EmpExprMapper.xml -->
<mapper namespace="com.itheima.mapper.EmpExprMapper">
    <!-- 批量插入工作经历 -->
    <insert id="insertBatch">
        insert into emp_expr (emp_id, begin, end, company, job) 
        values
        <foreach collection="exprList" item="expr" separator=",">
            (#{expr.empId}, #{expr.begin}, #{expr.end}, #{expr.company}, #{expr.job})
        </foreach>
    </insert>
</mapper>

// 对应的Mapper接口
@Mapper
public interface EmpExprMapper {
    void insertBatch(List<EmpExpr> exprList);
}
```

`<foreach>`标签核心属性说明：

|   属性名   |                 作用                 | 是否必填 |
| :--------: | :----------------------------------: | :------: |
| collection | 要遍历的集合名称（与方法参数名一致） |    是    |
|    item    |         遍历集合时的元素别名         |    是    |
| separator  |  元素之间的分隔符（批量插入用`,`）   |    否    |
|    open    |      遍历开始前拼接的 SQL 片段       |    否    |
|   close    |      遍历结束后拼接的 SQL 片段       |    否    |

#### （3）Service 层实现（含事务控制）

```java
@Service
public class EmpServiceImpl implements EmpService {
    @Autowired
    private EmpMapper empMapper;
    @Autowired
    private EmpExprMapper empExprMapper;

    // 事务注解：方法异常时回滚，保证数据一致性
    @Transactional(rollbackFor = Exception.class)
    @Override
    public void save(Emp emp) {
        // 1. 补全员工基本信息的公共字段
        emp.setCreateTime(LocalDateTime.now());
        emp.setUpdateTime(LocalDateTime.now());
        // 2. 插入员工基本信息（主键会自动回显到emp的id属性）
        empMapper.insert(emp);
        // 3. 批量插入工作经历（给每个经历设置empId）
        List<EmpExpr> exprList = emp.getExprList();
        if (!CollectionUtils.isEmpty(exprList)) {
            // 给每个工作经历关联当前员工ID
            exprList.forEach(expr -> expr.setEmpId(emp.getId()));
            empExprMapper.insertBatch(exprList);
        }
    }
}
```

#### （4）Controller 层实现

接收 JSON 格式参数（含工作经历列表）：

```java
@RestController
@RequestMapping("/emps")
@Slf4j
public class EmpController {
    @Autowired
    private EmpService empService;

    @PostMapping
    public Result save(@RequestBody Emp emp) {
        log.info("新增员工：{}", emp);
        empService.save(emp);
        return Result.success();
    }
}
```

## 二、事务管理

### 1. 事务核心概念

- 定义：事务是**一组不可分割的操作集合**，要么全部成功（提交），要么全部失败（回滚），避免数据不一致。
- 典型场景：银行转账、下单扣减库存、新增员工（关联多表操作）。

### 2. 事务基本操作（原生 SQL）

```java
-- 开启事务
start transaction; / begin;
-- 执行业务操作（如插入、更新、删除）
insert into emp (...) values (...);
insert into emp_expr (...) values (...);
-- 全部成功则提交事务
commit;
-- 任意一步失败则回滚事务（回到操作前状态）
rollback;
```

### 3. Spring 事务管理（注解式）

#### （1）核心注解：`@Transactional`

- 作用：将方法交给 Spring 管理事务，自动完成「开启事务→执行方法→提交 / 回滚」流程。

- 位置：

  - 方法上（推荐）：仅对当前方法生效
  - 类上：对类中所有方法生效
  - 接口上：对实现类的所有方法生效

  

#### （2）核心配置

```java
@Service
public class EmpServiceImpl implements EmpService {
    // rollbackFor = Exception.class：所有异常都回滚（默认仅RuntimeException回滚）
    @Transactional(rollbackFor = Exception.class)
    @Override
    public void save(Emp emp) {
        // 多表操作逻辑
    }
}
```

#### （3）关键属性

|   属性名    |                  作用                  |          默认值           |
| :---------: | :------------------------------------: | :-----------------------: |
| rollbackFor | 指定触发回滚的异常类型（如 Exception） | RuntimeException 及其子类 |
| propagation | 事务传播行为（多事务方法调用时的规则） |     REQUIRED（默认）      |

#### （4）事务传播行为（重点）

定义：当一个事务方法调用另一个事务方法时，如何控制事务（如加入现有事务、创建新事务）。

常用传播行为：

|    传播行为值    |                  含义                  |             应用场景             |
| :--------------: | :------------------------------------: | :------------------------------: |
| REQUIRED（默认） |    有事务则加入，无事务则创建新事务    |   大多数业务场景（如新增员工）   |
|   REQUIRES_NEW   | 无论有无事务，都创建新事务（相互独立） | 日志记录（无论主业务成败都执行） |

示例：新增员工时记录操作日志（日志独立事务）

```java
@Service
public class EmpServiceImpl implements EmpService {
    @Autowired
    private EmpLogService empLogService;

    @Transactional(rollbackFor = Exception.class)
    @Override
    public void save(Emp emp) {
        try {
            // 新增员工核心逻辑
        } finally {
            // 记录日志（独立事务，即使核心逻辑失败也执行）
            empLogService.insertLog(new EmpLog(...));
        }
    }
}

@Service
public class EmpLogService {
    // REQUIRES_NEW：创建新事务，与主事务独立
    @Transactional(propagation = Propagation.REQUIRES_NEW)
    public void insertLog(EmpLog empLog) {
        empLogMapper.insert(empLog);
    }
}
```

### 4. 事务四大特性（ACID）

|    特性     |                             含义                             |
| :---------: | :----------------------------------------------------------: |
| 原子性（A） |         事务是不可分割的最小单元，要么全成，要么全败         |
| 一致性（C） |    事务执行前后，数据总状态保持一致（如转账后总金额不变）    |
| 隔离性（I） |      多个事务并发执行时，互不干扰，避免脏读、幻读等问题      |
| 持久性（D） | 事务提交 / 回滚后，数据修改永久生效（即使数据库宕机也不会丢失） |

## 三、文件上传

### 1. 核心概念

文件上传：将本地文件（图片、视频、文档等）通过网络上传到服务器，供其他用户访问 / 下载，是 Web 开发常见功能（如头像上传、朋友圈发图）。

### 2. 前端核心要求（三要素）

```java
<!-- 表单上传示例 -->
<form action="/upload" method="post" enctype="multipart/form-data">
    头像：<input type="file" name="file">
    <input type="submit" value="上传">
</form>
```

- 提交方式：`method="post"`（文件数据较大，不适合 GET）
- 编码类型：`enctype="multipart/form-data"`（二进制流编码，支持文件传输）
- 文件控件：`type="file"`（用于选择本地文件）

### 3. 后端实现（SpringBoot）

#### （1）基础配置

默认单个文件上传大小限制为 1MB，需在`application.yml`中调整：

```yml
spring:
  servlet:
    multipart:
      max-file-size: 10MB  # 单个文件最大大小
      max-request-size: 100MB # 单次请求总大小（含多个文件）
```

#### （2）本地存储实现

将文件保存到服务器本地磁盘，适合小型项目：

```java
@RestController
@Slf4j
public class UploadController {
    // 本地存储路径（需提前创建目录）
    private static final String UPLOAD_DIR = "D:/images/";

    @PostMapping("/upload")
    public Result upload(MultipartFile file) throws Exception {
        log.info("文件上传：文件名={}, 大小={}KB", file.getOriginalFilename(), file.getSize()/1024);
        
        // 1. 生成唯一文件名（避免重名覆盖）
        String originalFilename = file.getOriginalFilename();
        String suffix = originalFilename.substring(originalFilename.lastIndexOf(".")); // 文件后缀（.jpg/.png）
        String uniqueFileName = UUID.randomUUID().toString().replace("-", "") + suffix; // 唯一文件名
        
        // 2. 保存文件到本地
        file.transferTo(new File(UPLOAD_DIR + uniqueFileName));
        
        // 3. 返回文件访问路径（本地存储需配置静态资源映射才能访问）
        return Result.success("/images/" + uniqueFileName);
    }
}
```

#### （3）阿里云 OSS 存储（推荐）

本地存储存在缺陷（磁盘满、无法跨服务器访问、易丢失），实际项目推荐使用云存储（如阿里云 OSS）。

##### 1. 阿里云 OSS 核心概念

- OSS：对象存储服务，海量、安全、高可靠的云存储服务，支持文件随时访问。
- Bucket：存储空间，用于存储文件（所有文件必须隶属于某个 Bucket）。
- AccessKey：访问密钥（AK/SK），用于身份认证，需妥善保管。

##### 2. 实现步骤

###### 第一步：准备工作

1. 注册阿里云并实名认证，开通 OSS 服务。
2. 创建 Bucket（设置存储类型、地域等）。
3. 获取 AccessKey（AK/SK）。

###### 第二步：引入依赖

```xml
<!-- 阿里云OSS SDK -->
<dependency>
    <groupId>com.aliyun.oss</groupId>
    <artifactId>aliyun-sdk-oss</artifactId>
    <version>3.15.1</version>
</dependency>
```

###### 第三步：配置文件参数（`application.yml`）

```yml
aliyun:
  oss:
    endpoint: https://oss-cn-beijing.aliyuncs.com # 地域节点（如北京）
    bucketName: java-ai # Bucket名称
    accessKeyId: your-access-key-id # 你的AK
    accessKeySecret: your-access-key-secret # 你的SK
    region: cn-beijing # 地域
```

###### 第四步：参数封装（`@ConfigurationProperties`）

批量注入配置文件参数，简化代码：

```java
@Data
@Component
@ConfigurationProperties(prefix = "aliyun.oss")
public class AliyunOSSProperties {
    private String endpoint;
    private String bucketName;
    private String accessKeyId;
    private String accessKeySecret;
    private String region;
}
```

###### 第五步：OSS 工具类实现

```java
@Component
@Slf4j
public class AliyunOSSOperator {
    @Autowired
    private AliyunOSSProperties ossProperties;

    // 上传文件到OSS，返回文件访问URL
    public String upload(byte[] fileContent, String originalFilename) throws Exception {
        // 1. 生成唯一文件名
        String suffix = originalFilename.substring(originalFilename.lastIndexOf("."));
        String fileName = UUID.randomUUID().toString().replace("-", "") + suffix;
        
        // 2. 初始化OSS客户端
        OSS ossClient = new OSSClientBuilder().build(
            ossProperties.getEndpoint(),
            ossProperties.getAccessKeyId(),
            ossProperties.getAccessKeySecret()
        );
        
        try {
            // 3. 上传文件（Bucket名称、文件路径、文件流）
            ossClient.putObject(
                ossProperties.getBucketName(),
                "avatar/" + fileName, // OSS中文件路径（如avatar/xxx.jpg）
                new ByteArrayInputStream(fileContent)
            );
            
            // 4. 拼接文件访问URL（公网可访问）
            return String.format("https://%s.%s/avatar/%s",
                ossProperties.getBucketName(),
                ossProperties.getEndpoint().replace("https://", ""),
                fileName
            );
        } finally {
            // 5. 关闭客户端
            ossClient.shutdown();
        }
    }
}
```

###### 第六步：上传接口开发

```java
@RestController
@Slf4j
public class UploadController {
    @Autowired
    private AliyunOSSOperator ossOperator;

    @PostMapping("/upload")
    public Result upload(MultipartFile file) throws Exception {
        log.info("OSS文件上传：{}", file.getOriginalFilename());
        // 调用工具类上传，获取文件URL
        String fileUrl = ossOperator.upload(file.getBytes(), file.getOriginalFilename());
        return Result.success(fileUrl); // 返回URL给前端，用于展示/存储
    }
}
```

### 4. 配置注入方式对比（`@Value` vs `@ConfigurationProperties`）

|          注入方式          |                 特点                 |       适用场景       |
| :------------------------: | :----------------------------------: | :------------------: |
|     `@Value("${key}")`     |        单个属性注入，语法简单        | 配置项较少（1-2 个） |
| `@ConfigurationProperties` | 批量注入，支持前缀匹配，类型自动转换 |  配置项较多、需复用  |

示例：

```java
// @Value注入（单个属性）
@Component
public class AliyunOSSOperator {
    @Value("${aliyun.oss.endpoint}")
    private String endpoint;
    @Value("${aliyun.oss.bucketName}")
    private String bucketName;
}

// @ConfigurationProperties注入（批量属性）
@Data
@Component
@ConfigurationProperties(prefix = "aliyun.oss")
public class AliyunOSSProperties {
    private String endpoint;
    private String bucketName;
    private String accessKeyId;
    private String accessKeySecret;
}
```

## 四、核心知识点总结

### 1. 新增员工关键技术

- 主键回显：`@Options(useGeneratedKeys = true, keyProperty = "id")`
- 批量插入：MyBatis`<foreach>`标签（遍历集合拼接 SQL）
- 多表关联：通过`emp_id`关联员工表和工作经历表

### 2. 事务管理

- 核心注解：`@Transactional`（方法 / 类 / 接口上）
- 关键属性：`rollbackFor`（指定回滚异常）、`propagation`（传播行为）
- 四大特性：ACID（原子性、一致性、隔离性、持久性）
- 常见传播行为：REQUIRED（默认）、REQUIRES_NEW（独立事务）

### 3. 文件上传

- 前端三要素：`post`方法、`multipart/form-data`编码、`file`控件
- 后端接收：`MultipartFile`对象（SpringBoot 自动封装）
- 存储方案：本地存储（小型项目）、阿里云 OSS（生产环境）
- 配置注入：多属性用`@ConfigurationProperties`，少属性用`@Value`

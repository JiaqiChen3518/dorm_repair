# Web 后端实战 - 员工管理（完结）笔记

## 一、删除员工功能

### 1. 需求分析

支持**单条删除**和**批量删除**统一接口，删除员工时需同时删除关联的工作经历，保证数据一致性。

- 核心逻辑：删除`emp`表数据 + 关联删除`emp_expr`表数据
- 事务控制：多表操作需加事务，要么同时成功，要么同时失败

### 2. 实现步骤

#### （1）Controller 层：接收参数

支持两种参数接收方式（数组 / 集合），请求路径：`DELETE /emps?ids=1,2,3

```java
@RestController
@RequestMapping("/emps")
@Slf4j
public class EmpController {
    @Autowired
    private EmpService empService;

    // 方式一：数组接收
    @DeleteMapping
    public Result delete(Integer[] ids) {
        log.info("批量删除员工：{}", Arrays.toString(ids));
        empService.deleteByIds(Arrays.asList(ids));
        return Result.success();
    }

    // 方式二：集合接收（推荐，无需手动转换）
    @DeleteMapping
    public Result delete(@RequestParam List<Integer> ids) {
        log.info("批量删除员工：{}", ids);
        empService.deleteByIds(ids);
        return Result.success();
    }
}
```

#### （2）Service 层：事务控制

```java
@Service
public class EmpServiceImpl implements EmpService {
    @Autowired
    private EmpMapper empMapper;
    @Autowired
    private EmpExprMapper empExprMapper;

    @Transactional(rollbackFor = Exception.class)
    @Override
    public void deleteByIds(List<Integer> ids) {
        // 1. 删除员工基本信息
        empMapper.deleteByIds(ids);
        // 2. 关联删除工作经历（emp_id in 员工ID列表）
        empExprMapper.deleteByEmpIds(ids);
    }
}
```

#### （3）Mapper 层：动态 SQL 批量删除

通过`<foreach>`标签拼接`in`条件，实现批量删除：

```java
<!-- EmpMapper.xml：批量删除员工 -->
<delete id="deleteByIds">
    delete from emp where id in
    <foreach collection="ids" item="id" separator="," open="(" close=")">
        #{id}
    </foreach>
</delete>

<!-- EmpExprMapper.xml：批量删除工作经历 -->
<delete id="deleteByEmpIds">
    delete from emp_expr where emp_id in
    <foreach collection="list" item="empId" separator="," open="(" close=")">
        #{empId}
    </foreach>
</delete>
```

`<foreach>`标签参数说明：

- `collection`：集合名称（方法参数为 List 时，默认`list`；指定参数名需用`@Param`）
- `open="("` + `close=")"`：拼接 SQL 的`in ()`结构
- `separator=","`：元素之间用逗号分隔

## 二、修改员工功能

### 1. 需求分析

分两步实现，保证数据完整性：

- 查询回显：根据员工 ID 查询基本信息 + 关联的工作经历
- 修改数据：更新员工基本信息 + 先删后加更新工作经历（避免重复）

### 2. 步骤 1：查询回显（关联多表查询）

#### （1）核心技术：MyBatis`resultMap`复杂结果封装

当实体包含**关联集合属性**（如`Emp`包含`List<EmpExpr>`），需通过`resultMap`手动封装：

```xml
<!-- EmpMapper.xml -->
<resultMap id="empResultMap" type="com.itheima.pojo.Emp">
    <!-- 员工基本属性映射 -->
    <id column="id" property="id" />
    <result column="username" property="username" />
    <result column="name" property="name" />
    <result column="gender" property="gender" />
    <!-- 其他基本属性（phone、job、entry_date等） -->
    <result column="entry_date" property="entryDate" />
    <result column="dept_id" property="deptId" />

    <!-- 关联工作经历集合：collection标签 -->
    <collection property="exprList" ofType="com.itheima.pojo.EmpExpr">
        <id column="ee_id" property="id" /> <!-- 别名避免字段冲突 -->
        <result column="ee_company" property="company" />
        <result column="ee_job" property="job" />
        <result column="ee_begin" property="begin" />
        <result column="ee_end" property="end" />
        <result column="ee_empid" property="empId" />
    </collection>
</resultMap>

<!-- 关联查询SQL：左连接查询员工+工作经历 -->
<select id="getById" resultMap="empResultMap">
    select
        e.*,
        ee.id ee_id,          <!-- 别名区分员工ID和工作经历ID -->
        ee.company ee_company,
        ee.job ee_job,
        ee.begin ee_begin,
        ee.end ee_end,
        ee.emp_id ee_empid
    from emp e
    left join emp_expr ee on e.id = ee.emp_id
    where e.id = #{id}
</select>
```

#### （2）Controller 层实现

```java
@GetMapping("/{id}")
public Result getInfo(@PathVariable Integer id) {
    log.info("根据ID查询员工详情：{}", id);
    Emp emp = empService.getInfo(id);
    return Result.success(emp);
}
```

### 3. 步骤 2：修改数据（先删后加）

#### （1）Service 层逻辑

```java
@Transactional(rollbackFor = Exception.class)
@Override
public void update(Emp emp) {
    // 1. 更新员工基本信息（补全更新时间）
    emp.setUpdateTime(LocalDateTime.now());
    empMapper.updateById(emp);

    // 2. 先删除该员工原有的工作经历
    empExprMapper.deleteByEmpIds(Arrays.asList(emp.getId()));

    // 3. 批量添加新的工作经历
    List<EmpExpr> exprList = emp.getExprList();
    if (!CollectionUtils.isEmpty(exprList)) {
        exprList.forEach(expr -> expr.setEmpId(emp.getId()));
        empExprMapper.insertBatch(exprList);
    }
}
```

#### （2）动态更新 SQL（`<set>`标签）

避免更新时多余逗号问题，`<set>`标签自动去除字段后的逗号：

```xml
<!-- EmpMapper.xml：动态更新员工信息 -->
<update id="updateById">
    update emp
    <set>
        <if test="username != null and username != ''">username = #{username},</if>
        <if test="name != null and name != ''">name = #{name},</if>
        <if test="gender != null">gender = #{gender},</if>
        <if test="phone != null and phone != ''">phone = #{phone},</if>
        <if test="job != null">job = #{job},</if>
        <if test="salary != null">salary = #{salary},</if>
        <if test="image != null and image != ''">image = #{image},</if>
        <if test="entryDate != null">entry_date = #{entryDate},</if>
        <if test="deptId != null">dept_id = #{deptId},</if>
        <if test="updateTime != null">update_time = #{updateTime}</if>
    </set>
    where id = #{id}
</update>
```

#### （3）Controller 层实现

```java
@PutMapping
public Result update(@RequestBody Emp emp) {
    log.info("修改员工信息：{}", emp);
    empService.update(emp);
    return Result.success();
}
```

### 4. `resultType` vs `resultMap`（重点）

|   映射方式   |                 适用场景                 |          特点          |
| :----------: | :--------------------------------------: | :--------------------: |
| `resultType` |  字段名与实体属性名完全一致，无复杂关联  |   简单、无需手动配置   |
| `resultMap`  | 字段名与属性名不一致，或实体包含关联属性 | 灵活、支持复杂结果封装 |

## 三、全局异常处理

### 1. 问题背景

传统异常处理通过`try-catch`包裹代码，存在**代码臃肿、重复度高、返回格式不统一**问题。

### 2. 解决方案：SpringBoot 全局异常处理器

通过`@RestControllerAdvice`+`@ExceptionHandler`实现全局异常捕获，统一返回格式。

#### （1）核心注解

- `@RestControllerAdvice`：组合注解（`@ControllerAdvice`+`@ResponseBody`），标识全局异常处理器
- `@ExceptionHandler`：指定捕获的异常类型（如`Exception.class`捕获所有异常）

#### （2）实现代码

```java
@Slf4j
@RestControllerAdvice // 全局异常处理器，作用于所有@RestController
public class GlobalExceptionHandler {
    // 捕获所有Exception类型异常
    @ExceptionHandler(Exception.class)
    public Result handleException(Exception e) {
        log.error("全局异常捕获：", e); // 打印异常堆栈信息（便于排查）
        // 统一返回错误结果（符合前端预期格式）
        return Result.error("服务器内部异常，请稍后重试");
    }

    // 捕获特定异常（如SQL唯一约束异常），优先级高于通用异常
    @ExceptionHandler(SQLIntegrityConstraintViolationException.class)
    public Result handleSQLException(SQLIntegrityConstraintViolationException e) {
        log.error("SQL异常：", e);
        String msg = e.getMessage();
        if (msg.contains("Duplicate entry")) {
            String username = msg.split("'")[1];
            return Result.error("用户名" + username + "已存在");
        }
        return Result.error("数据库操作异常");
    }
}
```

#### （3）优势

- 代码解耦：无需在各 Controller 重复写`try-catch`
- 格式统一：异常响应与正常响应格式一致（`Result`对象）
- 便于维护：集中处理所有异常，可按异常类型定制提示信息

## 四、员工信息统计

### 1. 核心需求

实现两类统计功能，返回前端可视化所需格式：

- 职位统计：统计各职位（班主任、讲师等）的员工人数
- 性别统计：统计男性、女性员工人数

### 2. 职位统计（SQL`case`函数）

#### （1）核心技术：SQL`case`流程控制函数

用于将数字编码转换为中文名称（如`job=1`→"班主任"），语法：

```sql
-- 语法1：条件匹配（适用于范围判断）
case when 条件1 then 结果1 when 条件2 then 结果2 else 默认结果 end

-- 语法2：等值匹配（适用于固定值映射）
case 字段 when 值1 then 结果1 when 值2 then 结果2 else 默认结果 end
```

#### （2）Mapper 层 SQL 实现

```xml
<!-- ReportMapper.xml -->
<select id="countEmpJobData" resultType="java.util.Map">
    select
        -- 将job数字编码转换为中文职位名
        (case when job = 1 then '班主任'
              when job = 2 then '讲师'
              when job = 3 then '学工主管'
              when job = 4 then '教研主管'
              when job = 5 then '咨询师'
              else '其他' end) as pos,
        count(*) as num -- 统计各职位人数
    from emp
    group by job -- 按职位分组
    order by num desc -- 按人数倒序
</select>
```

#### （3）Service 层封装结果

```java
@Service
public class ReportServiceImpl implements ReportService {
    @Autowired
    private ReportMapper reportMapper;

    @Override
    public JobOption getEmpJobData() {
        // 查询统计数据（List<Map>：每个Map包含pos=职位名，num=人数）
        List<Map<String, Object>> statList = reportMapper.countEmpJobData();
        
        // 解析数据封装为前端所需格式
        List<Object> jobList = statList.stream().map(map -> map.get("pos")).toList();
        List<Object> dataList = statList.stream().map(map -> map.get("num")).toList();
        
        return new JobOption(jobList, dataList);
    }
}

// 统计结果封装类
@Data
@NoArgsConstructor
@AllArgsConstructor
public class JobOption {
    private List jobList; // 职位列表（如["班主任","讲师"...]）
    private List dataList; // 人数列表（如[6,13...]）
}
```

#### （4）Controller 层实现

```java
@RestController
@RequestMapping("/reports")
public class ReportController {
    @Autowired
    private ReportService reportService;

    @GetMapping("/empJobData")
    public Result getEmpJobData() {
        JobOption jobOption = reportService.getEmpJobData();
        return Result.success(jobOption);
    }
}
```

### 3. 性别统计（SQL`if`函数）

#### （1）核心技术：SQL`if`函数

简化条件判断，语法：`if(表达式, 结果1, 结果2)`（表达式为 true 返回结果 1，否则返回结果 2）

#### （2）Mapper 层 SQL 实现

```java
<select id="countEmpGenderData" resultType="java.util.Map">
    select
        if(gender = 1, '男性员工', '女性员工') as name, -- 性别编码转中文
        count(*) as value -- 统计人数
    from emp
    group by gender -- 按性别分组
</select>
```

#### （3）Service+Controller 层实现

```java
// Service层
@Override
public List<Map> getEmpGenderData() {
    return reportMapper.countEmpGenderData();
}

// Controller层
@GetMapping("/empGenderData")
public Result getEmpGenderData() {
    List<Map> genderStat = reportService.getEmpGenderData();
    return Result.success(genderStat);
}
```

## 五、核心知识点总结

### 1. 删除员工

- 批量删除：MyBatis`<foreach>`标签拼接`in`条件
- 事务控制：多表删除需加`@Transactional`，保证数据一致性
- 参数接收：支持数组或`@RequestParam List<Integer>`接收批量 ID

### 2. 修改员工

- 复杂结果封装：`resultMap`+`collection`标签处理关联集合属性
- 动态更新：`<set>`标签自动去除多余逗号，适配部分字段更新
- 工作经历更新：先删除原有数据，再批量添加新数据（避免重复）

### 3. 异常处理

- 全局异常处理器：`@RestControllerAdvice`+`@ExceptionHandler`
- 优势：代码解耦、格式统一、支持特定异常定制处理

### 4. 统计功能

- SQL 流程控制：`case`函数（多条件匹配）、`if`函数（简单条件判断）
- 结果封装：通过 Stream 流解析`List<Map>`统计结果，适配前端可视化需求
